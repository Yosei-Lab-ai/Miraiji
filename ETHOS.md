# The Seller Ethos

gstack has the Builder Ethos. miraiji has the Seller Ethos.

Building is necessary but not sufficient. A product that nobody uses is a hobby project. There's nothing wrong with hobbies — but if you're trying to make a living, you need to sell.

---

## Core Beliefs

### Distribution is a skill, not a task

Marketing isn't something you do after the "real work" is done. It's a discipline with its own craft, its own feedback loops, and its own iteration cycles. Treat it with the same rigor you give your code.

### Price before you code

The price determines the architecture. A free tool doesn't need auth. A $10/mo SaaS needs Stripe integration, usage limits, and tier management. A $500/yr enterprise product needs seat-based licensing and an admin dashboard. Decide the price first. Then design the system.

### 100 users > 10,000 lines of code

A hundred people using your product tells you more than ten thousand lines of code ever will. Ship early, find users, listen to them. Your code is a hypothesis. Users are the experiment.

### The first 100 users are earned, not acquired

You don't "acquire" your first users through ads or growth hacks. You earn them — one conversation, one community post, one demo at a time. This is manual, unglamorous work. Do it anyway.

### Measure what matters, ignore vanity

Page views, GitHub stars, and Twitter impressions feel good but don't pay rent. Track the metrics that connect to revenue: signups, activation, retention, and willingness to pay.

### Solo-scale strategies only

If a strategy requires a 5-person marketing team, it's not a strategy for you. Every tactic miraiji recommends must be executable by one person in the time between shipping features.

---

## The Anti-Patterns

- **"I'll figure out pricing later"** — Later never comes. Price now.
- **"If I build it, they will come"** — They won't. Go find them.
- **"I'll just post it on HN"** — That's a lottery ticket, not a strategy.
- **"Everyone is my target user"** — Then no one is.
- **"I need to add more features first"** — No, you need more users first.

---

## The Pipeline Philosophy

gstack's pipeline ends at `/ship`. miraiji's pipeline starts there.

```
gstack:    Think → Plan → Build → Review → Test → Ship
                                                    ↓
miraiji:                                      Launch → Content → Outreach → Measure → Grow → Feedback → Retro
```

The code is the beginning, not the end.
